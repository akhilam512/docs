The Pipecat JavaScript SDK provides a lightweight client implementation that handles:

- Device and media stream management
- Managing bot configuration
- Sending actions to the bot
- Handling bot messages and responses
- Managing session state and errors

## Installation

Install the SDK and a transport implementation (e.g. Daily for WebRTC):

```bash
npm install @pipecat-ai/pipecat-js
npm install @pipecat-ai/realtime-ai-daily
```

## Quick Example

Here's a simple example using Daily as the transport layer:

```javascript
import { PipecatClient } from '@pipecat-ai/pipecat-js';
import { DailyTransport } from '@pipecat-ai/pipecat-daily';

// Handle incoming audio from the bot
function handleBotAudio(track, participant) {
  if (participant.local || track.kind !== 'audio') return;

  const audioElement = document.createElement('audio');
  audioElement.srcObject = new MediaStream([track]);
  document.body.appendChild(audioElement);
  audioElement.play();
}

// Create and configure the client
const client = new PipecatClient({
  params: {
    baseUrl: process.env.PIPECAT_API_URL || '/api',
  },
  transport: new DailyTransport(),
  enableMic: true,
  callbacks: {
    onTrackStart: handleBotAudio,
  },
});

// Connect to your bot
client.connect();
```

## Interaction Patterns

The SDK supports two main patterns for interacting with your bot:

### Real-time Connections

For voice and video chatbots, establish an ongoing connection using `connect()`. This enables:

- Live voice conversations
- Real-time video processing
- Continuous interactions

### Single-turn Actions

For simpler use cases, use `action()` to send individual requests:

- Text-to-text chat
- One-off image processing
- Batch operations
